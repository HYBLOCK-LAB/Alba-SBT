# Alba-SBT: 알바 경력 증명 시스템 PRD (Product Requirements Document)

**최종 수정**: 2026-04-29  
**버전**: 1.0  
**상태**: 개발 준비 완료

---

## 📋 Executive Summary

### 문제 정의

현재 알바생의 경력 증명 방식은 **파편화, 휘발성, 수동성** 3가지 치명적 문제를 가지고 있습니다.

| 문제점 | 상세 |
|--------|------|
| **파편화** | 고용보험, 통장 입금 내역, 가게 기록이 흩어져 있음 |
| **휘발성** | 4대 보험 미가입 알바는 사장님이 기억하지 않으면 기록이 사라짐 |
| **수동성** | 매번 서류를 직접 요청하고 발급받아야 함 |

또한 플랫폼에서는 **허위 경력 작성, 경력 부풀리기, 경력 세탁** 등으로 인한 신뢰 붕괴 문제가 심각합니다.

### 솔루션

**블록체인 기반 SBT(Soulbound Token) + EAS(Ethereum Attestation Service)를 활용한 자동 경력 증명 시스템**

- ✅ **데이터 기반**: QR, GPS를 통해 객관적 데이터 자동 수집
- ✅ **자동 발행**: 조건 달성 시 EAS가 자동 발행, 사장님 개입 없음
- ✅ **온체인 기록**: 모든 증명이 블록체인에 영구 기록되어 조작 불가
- ✅ **검증 가능**: 누구나 언제든 EAS Explorer로 상세 데이터 추적 가능

## 📱 제품 범위

### 1️⃣ 알바생 앱 (React Native)

**홈 — 섹션 2개**
- 매장 연결: 여러 매장 코드 등록 → 사장님 승인 완료 매장만 목록 활성화
- 경력 프로필 대시보드: 현재 레벨(SBT 배지 시각화), 레벨업 프로그레스 바, 포트폴리오 공유 QR 버튼

**매장별 랜딩 — 탭 3개**
- QR 스캔: 출/퇴근 등록 (GPS 또는 매장 QR 검증, 불일치 시 거부)
- 근태 관리: 월간 캘린더, 실시간 근무 현황(예약 퇴근 기준 남은 시간)
- 신청: 스케줄 변경, 연장근무(익월까지)

### 2️⃣ 사장님 앱 (React Native)

**홈 - 탭 2개**
- 탭 1️⃣ 매장 관리 (등록된 매장 목록 + 매장 등록하기 버튼)
- 탭 2️⃣ 지원자 경력 검증 (포트폴리오 QR 스캔 → EAS/SBT 신뢰도 리포트 즉시 조회)

**매장별 랜딩 - 탭 4개**
- 인사 관리: 알바 등록(매장코드 발급→알바생 입력→사장 최종 승인) / 삭제 / 상세 조회
- 근태 관리: 캘린더 일정, 금일 출근 현황표, 근무일 변경 및 연장근무 승인 요청 처리
- 출퇴근 QR 생성: [QR 생성] 버튼 누르면 30초간 QR 표시 → 30초 후 자동 소멸 / 재생성하려면 버튼 재클릭 / 태블릿 앱고정 운영
- 승급 승인: 조건 충족 시 자동 리스트 등록 → Multi-Signature 서명 승인

## 🎬 사용자 흐름 예시

### 알바생 관점

```
1. 앱 → 지갑 연동 (MetaMask)
2. 홈 → [매장 연결하기] → 매장코드 입력 → 사장님 승인 대기
3. 승인 완료 → 매장 카드 활성화, 매장별 랜딩 접근 가능
4. 출근: 매장 랜딩 → [QR 스캔 탭] → [출근 QR 스캔] → GPS/QR 검증 → 출근 기록
5. 실시간: 근태 관리 탭에서 예약 퇴근까지 남은 시간 확인
6. 퇴근: QR 스캔 탭 → [퇴근 QR 스캔]
7. 필요 시: 신청 탭에서 스케줄 변경 / 연장근무 신청
8. 6개월 경과: EAS_EXP_TIME 자동 발행 → 대시보드 업데이트
9. 승급 조건 충족 → 사장님 승인 → Multisig → SBT 발급 → 배지 추가
10. 새 일자리 지원: 홈 대시보드 → [포트폴리오 공유 QR] 제시
```

### 사장님 관점

```
1. 앱 → 지갑 연동 (MetaMask)
2. 매장 등록:
   - 매장명, 카테고리(대분류 → 하위 업종), 위치(GPS), 사업자번호(선택), 연락처 입력
   - 매장코드 6자리 자동 생성
3. 알바생 관리:
   - 생성된 매장코드를 알바생에게 공유
   - 알바생이 앱에 코드 입력 → 사장 앱에서 최종 승인 (오프체인)
4. 일상 운영:
   - 매일: [QR 생성] 버튼으로 당일 매장 고유 QR 생성 → 30초간 유효 후 자동 소멸 → 필요 시 버튼 재클릭으로 재생성
   - 캘린더에서 월간 근태 현황 + 실시간 출근 현황 확인
   - 근무 일정 변경 / 연장근무 승인 요청 처리
5. 승급 승인:
   - 직원의 승급 판독 완료 → 승인 리스트에 자동 등록
   - 각 알바생 선택 후 [승인] → EIP-712 오프체인 서명 → SBT 발급
6. 지원자 검증:
   - 지원자의 포트폴리오 공유 QR 스캔
   - EAS/SBT 신뢰도 리포트 즉시 조회 → 채용 판단
```

## 🔐 주요 기능 상세

### 알바생 앱

#### 홈

**섹션 1: 매장 연결**
- **[매장 연결하기] 버튼**: 매장코드(6자리) 입력 → 사장님 승인 요청 전송
- **연결된 매장 카드 목록**: 승인 완료 매장만 활성화 / 미승인은 "승인 대기" 표시
- **카드 탭**: 해당 매장의 랜딩 페이지로 이동

**섹션 2: 경력 프로필 대시보드**
- **현재 레벨 + SBT 배지 시각화**: 획득한 배지 이미지를 그리드 형태로 표시
- **레벨업 프로그레스 바**: 현재 보유 EAS 개수 vs 다음 레벨 필요 요건 비율을 % 진행률로 표시
- **[포트폴리오 공유 QR] 버튼**: wallet_address 인코딩 QR 생성 (채용처 검증용)

#### 매장별 랜딩 — 탭 3개

**탭 1: QR 스캔**
- **[출근 QR 스캔] / [퇴근 QR 스캔] 버튼**
- **검증 로직 (필수 구현)**:
  - GPS 좌표 ↔ 매장 등록 좌표 반경 50m 이내 대조
  - OR 사장님 앱에서 생성한 당일 매장 고유 QR 일치 확인
  - 두 조건 모두 불일치 시: "위치/QR이 일치하지 않습니다" 경고 후 출퇴근 처리 거부
- **출퇴근 기록**: 검증 통과 시 `attendance` 테이블에 저장 (clock_in, clock_out, gps_coords, shop_id)

**탭 2: 근태 관리**
- **월간 근태 현황 캘린더**: 정상 출근(초록), 지각(노랑), 결근(빨강) 색상 구분
- **실시간 근무 현황**: clock_in 완료 후 예약 퇴근 시간까지 남은 시간 카운트다운 표시

**탭 3: 신청**
- **스케줄 변경 신청**: 근무 날짜/시간 변경 요청 → 사장님 앱에서 승인/거부 처리
- **연장근무 신청**: 현재 월 + 익월까지 신청 가능 → 사장님 앱에서 승인/거부 처리


### 사장님 앱

#### 홈 — 탭 2개

**탭 1: 매장 관리**
- **매장 목록**: 등록된 매장 표시 (매장명, 카테고리, 최근 출근 현황)
- **매장 등록하기 버튼**: 새 매장 추가 시 클릭
  - 입력 필드: 매장명, 카테고리(대분류 → 하위 업종), 위치(GPS), 사업자번호(선택), 연락처
  - 매장코드 6자리 자동 생성 (예: 103078)
  - 매장코드 복사 버튼으로 알바생 공유 용이

**탭 2: 지원자 경력 검증**
- **포트폴리오 QR 스캔**: 지원자의 wallet_address QR 스캔
- **신뢰도 리포트 즉시 조회**:
  - 보유한 SBT 레벨 및 모든 EAS 목록 표시
  - 예: "[경험] F&B 3년 | [성실성] 지각 0회 (6개월 × 2회) | [신뢰도] 스케줄 변경 0회 | Lv.6 Expert"
  - 블록체인에서 직접 조회 (실시간, 조작 불가)

#### 매장별 랜딩 — 탭 4개

**탭 1: 인사 관리**
- **알바 등록**: 생성된 매장코드를 알바생에게 공유 → 알바생이 앱에 입력 → 사장 승인 (오프체인)
- **직원 리스트**: 등록된 알바생 표시 (이름, 현재 등급, 상세 조회 버튼)
- **직원 삭제**: 선택 후 삭제 처리

**탭 2: 근태 관리**
- **일정 캘린더**: 전체 알바생의 월간 근무 현황 (색상 구분)
- **금일 출근 현황표**: 실시간 출근/미출근 현황 (색상: 출근=초록, 미출근=회색, 결근=빨강, 지각=노랑)
- **근무일 변경 승인**: 알바생이 요청한 일정 변경 건 처리
- **연장근무 승인**: 알바생이 요청한 연장근무 건 처리

**탭 3: 출퇴근 QR 생성**
- **[QR 생성] 버튼**: 누르면 당일 매장 고유 QR 표시 → 30초 후 자동 소멸
- **30초 카운트다운**: QR 표시 중 남은 유효 시간 표시
- **재생성**: QR 소멸 후 버튼을 다시 눌러야 새 QR 생성 (자동 갱신 없음)
- **유효 시간 설정**: 예: 08:00 ~ 23:00 (설정 범위 외 스캔 시 거부)
- **태블릿 앱고정**: 매장의 태블릿에 QR을 계속 노출 (가능하도록 UI 최적화)

**탭 4: 승급 승인**
- **승급 조건 충족 리스트**: 백엔드 판독에서 조건 달성한 알바생만 자동 노출
- **각 알바생 선택 후 [승인] 버튼 클릭**:
  - 지갑 연동 → EIP-712 오프체인 서명 (가스비 0)
  - 플랫폼이 자동으로 2번째 서명 추가
  - 스마트 컨트랙트 전송 → SBT 자동 발급

## 🔗 블록체인 설계 — EAS + SBT 결합 모델

### 핵심 개념

| 레이어 | 기술 | 역할 | 발행 시점 |
|--------|------|------|---------|
| **하부 (팩트)** | EAS | 세부 근태·업무 증명 | 조건 달성 시 자동 |
| **상부 (배지/레벨)** | SBT (ERC-5192) | 최종 커리어 배지 | 레벨 승급 시만 |

### EAS Building Blocks (추가 및 수정 가능)

각 항목이 조건을 충족할 때마다 **독립적인 EAS가 온체인 자동 발행**됩니다.

| EAS 유형 | 이름 | 발급 트리거 | 데이터 필드 | 재발급 |
|----------|------|------------|-----------|--------|
| `EAS_EXP_TIME` | 업종 경력 증명 | 동일 업종 누적 6개월 | 업종, 근무지, 기간 | ✅ 반복 가능 |
| `EAS_FAITH_ATT` | 성실성 증명 | 3개월 지각·결근 0회 | 출근율, 기간 | ✅ 반복 가능 |
| `EAS_SCHED_RELI` | 신뢰도 증명 | 3개월 스케줄 변경 0회 | 변경 횟수, 점수 | ✅ 반복 가능 |

## 🔄 전체 데이터 파이프라인 (5단계)

### Step 1️⃣ — Raw Data Collection

| 데이터 항목 | 판정 기준 | DB 컬럼 | EAS 연결 |
|-------------|----------|---------|---------|
| **출석** | 예약 출근 시간 ± 허용 범위(예: 10분) 내 clock_in | `attendance.status = 'on_time'` | `EAS_FAITH_ATT` |
| **지각** | 예약 출근 시간 초과 후 clock_in | `attendance.status = 'late'` | `EAS_FAITH_ATT` |
| **결근** | 근무 예정일에 clock_in 없음 (자정 기준 판정) | `attendance.status = 'absent'` | `EAS_FAITH_ATT` |
| **근무 기간** | 특정 매장의 첫 번째 `clock_in` ~ 마지막 `clock_in` 또는 `clock_out` 사이 기간 (월 단위 환산) | `attendance` 테이블에서 집계: `MIN(clock_in)` ~ `MAX(COALESCE(clock_out, clock_in))`, `store_id` 기준 | `EAS_EXP_TIME` |
| **업종** | 매장 등록 시 설정 (F&B / 유통·물류 / 서비스 / 기타) | `stores.category` | `EAS_EXP_TIME` |
| **스케줄 변경 횟수** | 확정된 근무 일정을 근무 당일 N시간 이전에 변경 요청한 횟수 | `schedule_changes` 테이블 COUNT | `EAS_SCHED_RELI` |
| **연장근무 시간** | 승인된 연장근무 건의 누적 시간 합산 | `attendance` 테이블에서 `type='extension'` 집계: `SUM(extension_hours)`, `store_id` 기준 | `EAS_SUB_SUPPORT` |

---

### Step 2️⃣ — EAS Attestation (팩트 증명 자동 발행)

- Supabase Edge Functions가 **주기적으로** DB 스캔
- EAS 발행 요건 충족 여부 확인 (예: 6개월 경과, 지각 0회 등)
- 조건 충족 시 백엔드가 **플랫폼 서명** → EAS SDK로 Sepolia 네트워크에 증명서 온체인 발행
- **사장님 개입 없음**: 객관적 데이터에 기반한 완전 자동 발행

---

### Step 3️⃣ — Level-up Detection (승급 판독)

- PM이 구현한 **승급 판독 엔진**이 유저 지갑 주소에 연결된 모든 EAS UID 조회
- 복합 조건 비교 (예: "Lv.4 승급 위해 서로 다른 업종 `EAS_EXP_TIME` 3개 보유 여부")
- 조건 충족 시 `Pending Level-up` 상태로 전환
- Multisig 절차 준비

---

### Step 4️⃣ — Multisig 서명 프로세스

- **사장님 서명 (1/2)**: 가스비 없는 EIP-712 오프체인 서명
  - 내용: "이 알바생의 승급 데이터가 사실임을 보증합니다"
  
- **백엔드 서명 (2/2)**: 서버가 사장님 서명값 + 시스템 서명값 묶음

- 완성된 두 서명을 스마트 컨트랙트로 전송

---

### Step 5️⃣ — SBT 민팅

- Solidity 컨트랙트가 `ecrecover`로 두 서명 검증
- 검증 통과 시 ERC-5192 SBT를 유저 지갑으로 발행
- SBT 메타데이터(JSON) → IPFS/Supabase Storage 저장 → `tokenURI` 연결

---

### Step 6️⃣ — Portfolio & Verification

- 알바생 앱의 SBT 지갑에서 획득 배지 시각적 표시
- 다른 사장님이 알바생의 공유 QR 스캔 → EAS Explorer/Etherscan으로 SBT 뿌리 EAS 데이터 즉시 역추적 가능

**승인 흐름 (C + B-2 연계)**

```
사장님이 [승인] 버튼 클릭 (C가 구현한 화면)
        ↓

        ↓
사장님 MetaMask에서 "확인"
        ↓
C: 서명값을 백엔드(B-2)로 전송
        ↓
B-2: 플랫폼 서명 추가 후 C가 만든 컨트랙트 호출
        ↓
컨트랙트: 두 서명 검증 통과 → SBT 발행
        ↓
C: 알바생 앱 배지 대시보드에서 ABI로 SBT 조회 → 배지 표시
```

## 🔍 주요 논의사항

### EAS_SVC_COUNT (서비스 숙련도 증명) 보류

**원래 의도**
- 결제/제조 횟수를 통해 "얼마나 숙련했는가"를 객관적으로 입증
- POS 시뮬레이터에서 알바생이 처리한 거래 자동 카운팅

**현재 문제점**

1. **업종별 데이터 다양성**
   - F&B: 음료 제조 횟수, 주문 처리 건수
   - 유통·물류: 배달 횟수, 하역량
   - 서비스: 고객 응대 횟수, 상담 시간
   - → 동일한 기준으로 숙련도를 비교하기 어려움

2. **객관적 수치 측정 불가**
   - 서비스 파트(안내 데스크, 고객 상담): 정량화 불가능
   - POS 기반 카운팅이 실제 숙련도를 반영하지 못할 수 있음

3. **UX 부담**
   - 알바생이 매 결제/제조마다 POS에 자신의 정보 등록
   - 실제 현장에서 번거롭고 오류 가능성 높음

**현재 상태**
- Phase 1(백엔드) 개발 범위에서 제외
- 4개 EAS (경력, 성실성, 신뢰성, 협력도)로 핵심 가치 충분히 제공
- 향후 업종별 특화 EAS 개발 시 재도입 예정

**향후 재도입 전략**
- 업종별로 "숙련도"의 정의를 명확히 (예: "F&B 제조 숙련도" vs "유통 배달 숙련도")
- 각 업종에 맞는 EAS 설계 (EAS_BARISTA_SKILL, EAS_DELIVERY_COUNT 등)
- UX 개선: 자동 카운팅 또는 사장님이 한 번에 월 실적 입력
- Phase 4에서 본격 개발

**필요한 팀 논의**
- "숙련도 없이 신뢰성만으로도 충분한가?" → **결론: 충분함**
  - 사장님 관점: "일을 많이 했는가"보다 "확실히 올 사람인가"가 더 중요
  - 신뢰·성실성으로 "믿을 수 있는 사람" 입증 가능
  - 양적 숙련도는 나중에 추가해도 됨
